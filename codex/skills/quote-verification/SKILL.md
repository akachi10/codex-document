# 引文核验（quote-verification）

## 铁律

1. **引文必须真实存在**：能定位到权威原典（具体书卷/章节/版本）才可用
2. **逐字准确**：与原典逐字对照，一个词都不许改。禁止"改写得更顺""拼接两句成一句""换个更美的词"
3. **版本锁定**：同一文本的不同译本不可混用（如意语 Diodati ≠ 英语 KJV ≠ 中文和合本各是独立文本）；页面标注出处时写明版本
4. **核不到就不用**：找不到可靠原典的"名言"一律弃用，换成零修辞的老实事实陈述——用户明确偏好"老实事实"胜过 AI 造的精心句

## 核验流程

1. **定位原典**：用 Codex web search / `open` 找权威源（圣经用官方在线原典站点、文学用权威电子文库；避免名言聚合站——它们本身就是以讹传讹重灾区）
2. **逐字对照**：把候选引文与原典逐字 diff，任何差异都改回原典
3. **多语场景**：每个语种分别对照该语种的权威译本，禁止自己翻译经典文本
4. **登记出处**：交付物中注明 书卷 章:节 / 作品+章节 / 版本名
5. **派单场景**：涉及引文的任务派给 agent 时，派单 prompt 里必须写明本铁律 + 指定权威源；agent 交付后 session 抽查对照

## 反面模式（都发生过）

- 把两节经文各取半句拼成"新句子"
- 用记忆里的近似措辞代替原文
- 为排版好看删掉引文中的"多余"词
- 编造"某某旗下""某某曾说"类事实性表述——法律文档的主体/联系方式属"事实"，先问用户，不从记忆/域名猜

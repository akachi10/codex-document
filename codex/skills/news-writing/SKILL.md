# news-writing — 中文新闻文体 + 反 AI 味

中文新闻文体的两条命：**事实归事实，观点归观点 + 不要 AI 味**。

## 何时使用

- 写正文（持续遵循，不是写完才看）
- 改稿 / 润色
- 拟标题
- 提交前最后过一遍 checklist

## 核心规则：事实与观点严格分离

### 事实段

只写**可验证的事实**：

- 时间、地点、当事人、数字、引述原文
- 不夹带评价 / 形容词倾向 / 立场暗示
- 写法：「4 月 25 日，A 公司宣布以 120 亿美元收购 B 公司。」（OK）
- 反例：「4 月 25 日，A 公司**宣告**以**惊人的** 120 亿美元**强势**收购 B 公司。」（不 OK，"宣告 / 惊人 / 强势"都带情绪）

### 引述他人观点

必有**归因 + 出处**：

- 「路透社 4 月 26 日报道，反垄断审查可能耗时 12 个月以上。」
- 「公司 CEO 张三在 4 月 28 日财报电话会议表示：「这次并购对客户是好事。」」
- 反例：「业内人士认为审查会很久。」（"业内人士"是匿名万金油，**禁止**）

### 主编自己的观点

**必须独立成段**，段首加 `**主编观点：**` 标签：

```
**主编观点**：在云服务高度集中的当下，再增加一笔横向并购对中小客户议价
能力是负面信号，但最终影响仍取决于 FTC 的救济条件。
```

主编观点段：
- 一篇文章最多 1-2 段
- 必须基于已核查的事实，不凭空发议论
- 不预测市场走势 / 不给投资建议
- 可以表达倾向，但要克制

## 标题规则

| 项 | 要求 |
|---|---|
| 句式 | 陈述句 > 问句；**禁止**感叹句 |
| 标点 | **不**用感叹号；**慎**用问号；句号一般省略 |
| 情绪词黑名单 | 震惊 / 突发 / 重磅 / 速看 / 必看 / 万万没想到 / 神反转 / 一夜变天 / 史诗级 / 史无前例 |
| 夸大警报 | 「罕见 / 巨变 / 全网 / 引发热议」需要事实支撑，没数据不要用 |
| 长度 | 中文 14-26 字，英文 8-15 词为佳 |
| 内容 | 标题必须如实概括正文核心；和正文不一致 = 标题党 |

**好标题**：
- 「OpenAI 发布 GPT-5，文本生成基准较上代提升 23%」
- 「美联储 4 月会议纪要：多数官员支持暂缓降息」

**坏标题**：
- 「震惊！OpenAI 重磅发布最新模型」（情绪词 / 感叹）
- 「美联储这次又要变天了！」（夸大 / 感叹 / 没说什么）

## 段落规则

- 80-150 字一段
- 一段一个核心信息
- 段间留空行
- 全文 3-6 段（快讯 1-3 段，长读 8-15 段）

## 引用规则

- 引述原文用「」（中文方括号），不用 ""（英文双引号）或 '"
- 长引用单独成段，加引号 + 出处行：

```
「我们对未来 12 个月的需求保持谨慎乐观。」
—— 公司 CFO 李四，4 月 28 日财报电话会议
```

- 短引用嵌入正文：「公司称将在 Q3 推出新产品。」
- 引用必须**逐字**复制，不能转述后加引号
- 编辑过的引用（截短 / 改语序）用 [...] 或 [中略] 标记

## 数字与日期

| 项 | 写法 |
|---|---|
| 日期 | 完整年月日：「2026 年 4 月 28 日」；行内可省年份：「4 月 28 日」 |
| 时间 | 24 小时制：「14:30」；跨时区注明：「14:30 UTC」 |
| 金额 | 标币种：「120 亿美元 / 850 亿元人民币」；不混用 |
| 百分比 | 说明同比 / 环比 / 较去年同期 |
| 大数字 | 中文用「亿 / 万」，英文用「billion / million」；不混用 |
| 区间 | 「23-25 美元」用半角连字符，不用波浪号 |

## 中文 AI 味黑名单（持续维护）

### 禁用词

| 类别 | 词 |
|---|---|
| 商业大词 | 赋能 / 闭环 / 抓手 / 生态 / 范式 / 矩阵 / 心智 / 飞轮 / 链路 / 中台 |
| 万能形容词 | 强大 / 全方位 / 无缝 / 智能化 / 深度 / 高效 / 一站式 / 高质量 |
| 情绪词 | 震惊 / 重磅 / 突发（除真突发新闻）/ 火爆 / 神级 |
| 网络梗 | yyds / 绝绝子 / 卷 / 躺平（除主题就是这些）/ 整顿职场 |
| 学术黑话 | 解构 / 范式转移 / 后现代 / 元叙事（除非真讲哲学）|
| 万金油 | 业内人士 / 据悉 / 有消息称 / 网传（无具体来源时） |

### 禁用句式

- 「让我们一起... / 让我们来看看...」
- 「在如今... 的时代 / 在 X 大背景下」
- 「...无疑是...的最佳选择」
- 「...到底意味着什么？」（标题党）
- 「值得我们深思」
- 「这背后的故事」
- 「不容错过」

### 替换建议

| AI 味 | 改写 |
|---|---|
| AI 赋能企业 | AI 工具用于企业（具体说什么工具、什么场景） |
| 重磅发布 | 4 月 28 日发布 |
| 行业大佬 | 具体人名 + 头衔 |
| 业内人士透露 | 删除或写"X 公司一位匿名员工告诉路透社" |
| 一站式解决方案 | 具体功能列表 |
| 让我们一起来看 | 直接写事实 |

## 立场表达

- 事实段：**禁止**夹带立场
- 引用他人：必有归因
- 主编观点：独立段 + 标签
- **禁止**用「显然 / 不言而喻 / 众所周知」（这些是回避论证）
- **禁止**用「我们 / 我们认为」（用「主编观点：」段代替；事实段不带主语）

## 结尾

- 不强行总结
- 不预测
- 可写「待观察的下一步信号」 / 「关键时间节点」
- 财经稿件常见结尾：「公司将于 X 月 X 日发布下一季度业绩。」
- 科技稿件常见结尾：「该模型 / 协议 / 项目目前在 X 阶段，下一步是 Y。」

## 自检（提交前过一遍）

完稿后逐项打勾：

- [ ] 标题没有感叹号、没有"震惊 / 重磅"等情绪词
- [ ] 标题如实概括正文，没有夸大
- [ ] 事实段没有主观形容词
- [ ] 所有引用都有出处
- [ ] 没有「业内人士 / 据悉 / 网传」等万金油
- [ ] 主编观点段独立成段且有 `**主编观点：**` 标签
- [ ] 数字、日期、币种格式规范
- [ ] AI 味黑名单词全部检索后清理
- [ ] 段落 80-150 字，整体 3-6 段（快讯）/ 8-15 段（长读）
- [ ] 没有强行总结或预测
- [ ] 引语用「」不用 ""

任一项不达标 → 改。改完再读一遍。

## 反例（什么时候不要用本 skill）

- 还在选题阶段（用 `news-pitch`）
- 还没核对信源（用 `source-verification`）
- 已经写完准备发布前的事实核查（用 `fact-checking`）
- 写代码 / 写文档（不是新闻稿，不适用）

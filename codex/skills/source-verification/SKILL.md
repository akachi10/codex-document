# source-verification — 信源核验

写作中遇到要引用的内容，先做信源体检。**没体检不准引用**。

## 何时使用

- 准备引用某条 Tweet / 微博 / TG 帖子
- 准备引用某篇外媒报道
- 准备引用某张图片 / 某段视频
- 准备引用某份数据 / 某个图表
- 准备引用某个个人或机构的发言

## SIFT 四步法

每条引用都过这四步。**先 Stop，再行动。**

### S — Stop

先别复制粘贴。问自己：
- 这条信源我熟悉吗？
- 转发链是什么？最早是谁发的？
- 我是不是因为这条内容"符合我的预期"才想引用？（确认偏误警报）

### I — Investigate the source

调查信源本身：

| 信源类型 | 调查项 |
|---|---|
| 个人账号 | 注册时间、历史发帖、蓝标 / 实名信息、其他平台账号是否一致、过往可信度 |
| 媒体账号 | 是否官方认证、所属机构、过往报道质量、是否有撤稿记录 |
| 机构账号 | 是否官方域名、是否有备案 / 注册信息、域名 WHOIS |
| 公司公告 | 是否官方网站 / 投资者关系页 / 监管平台（SEC EDGAR / 巨潮资讯 / HKEX）|
| 学术论文 | 是否经同行评审、期刊影响力、作者背景、是否预印本（arXiv / SSRN） |

### F — Find better coverage

找更好的覆盖：

- 这件事**有没有主流媒体也在报**？路透 / 彭博 / FT / 纽时 / 华尔街日报 / 澎湃 / 财新等
- 多家口径**一致**才能引；如果只有一家在报、其他没跟，先标"目前仅 X 报道"
- 如果是政府 / 公司行为，回到**一手发布渠道**（官网 / 官方公告 / 监管平台）核对原文

### T — Trace claims to original

追溯到原始来源：

- 二手报道 → 找一手发布
- 截图 → 找原帖 URL
- 数据 → 找原始数据集 / 报告 PDF
- 图片 → 反向图搜（Google Lens / TinEye / Yandex 图搜）
- 视频 → InVID / WeVerify 拆帧反查 / 看 metadata

## 信源信誉评级

引用前给每个信源打分：

| 等级 | 标准 | 处理 |
|---|---|---|
| **高** | 一手权威（监管文件 / 官方公告 / 同行评审论文 / 当事人本人发布） | 可直接引用 |
| **中** | 主流大型媒体的署名报道，与至少 1 个独立来源交叉印证 | 可引用，需标明出处和发布时间 |
| **低** | 单一媒体报道无交叉、二手转述、未署名爆料 | 慎用，必须明示"目前仅 X 报道，本文无独立验证" |
| **不可用** | 匿名爆料 / 私人 TG 群截图 / 未实名小号 / 内容农场 | **不要引用** |

## 社交账号验真 checklist

引用 Tweet / 微博 / TG / Bluesky 帖子前过一遍：

- [ ] 账号注册时间 > 1 年（新号警报，可能是钓鱼或临时号）
- [ ] 历史发帖语言风格、发帖时区一致（不一致警报）
- [ ] 其他平台是否有同名一致的账号
- [ ] 蓝标 / 实名 / 机构认证状态
- [ ] 帖子未被作者本人删除（截图后帖子可能被删）
- [ ] 截图与原帖 URL 比对，文字、时间戳、互动数一致

## 反向图搜（Reverse Image Search）

引用图片前**必须**反向搜：

| 工具 | URL | 用途 |
|---|---|---|
| Google Lens | https://lens.google.com/ | 全球覆盖广，识别商品和地点强 |
| TinEye | https://tineye.com/ | 找到图片首次出现时间 |
| Yandex 图搜 | https://yandex.com/images/ | 在中俄内容上比 Google 更强 |

**找出图片：**
1. 第一次出现的时间和站点
2. 是否被 P 过 / 是否是合成
3. 是否被错误情境化（旧图新发）

EXIF 元数据：用 ExifTool 或在线工具读拍摄时间 / 设备 / GPS（注意社交平台通常会剥 EXIF）。

## 视频验证

| 工具 | 用途 |
|---|---|
| **InVID** | 浏览器插件，关键帧提取 + 反向图搜 |
| **WeVerify** | InVID 的衍生项目，更多元数据 |
| **YouTube DataViewer** | 通过 YouTube ID 提取视频上传时间 |

验证项：上传时间、地点是否一致（GPS / 地标 / 路牌）、画面与音频是否同步、是否有剪辑痕迹。

## 合法访问 paywall

遇到付费墙文章是常见情况。合法绕过工具：

| 工具 | 合法性 | 用法 |
|---|---|---|
| **Unpaywall** | 完全合法 | 浏览器插件，自动找该论文 / 文章的开放访问版本 |
| **图书馆数据库** | 合法 | 大学 / 公共图书馆订阅，VPN 进图书馆 IP |
| **作者预印本** | 合法 | 论文作者通常在 arXiv / SSRN / 个人主页放预印版 |
| **机构官网** | 合法 | 公司 / 政府公告通常在自家官网免费公开 |
| **Internet Archive** | 灰区 | Wayback Machine 上的快照可看，但通常不能下载 PDF |

**不合法的工具不要碰**（Sci-Hub / 12ft.io 在多国管辖下不合法 / 不稳定）。

## 信源体检表

每篇文章生成一份 `docs/maintainer/source-checks/{article-slug}.md`（或项目 persona 指定的等价路径）：

```markdown
# Source Checks: <文章标题>

- **关联文章**：<pitches/...md 的 slug>
- **核验时间**：<YYYY-MM-DD HH:MM>

## 引用清单

### 引用 1
- **正文位置**：<段落起始字句>
- **信源**：<URL / 名称>
- **信源类型**：<个人 / 媒体 / 机构 / 公告 / 论文 / 图 / 视频>
- **SIFT 走完了**：S [ ] I [ ] F [ ] T [ ]
- **信誉评级**：高 / 中 / 低 / 不可用
- **交叉印证来源**：<URL>
- **备注**：<疑点 / 待确认项>

### 引用 2
（同上）

## 总结

- 全部引用至少有几个高 / 中评级？
- 是否所有「低」评级的引用在正文里都做了"目前仅 X 报道"的限定？
- 是否有「不可用」评级被引用？（必须移除）
```

## 自检

体检完成后回答：

1. 每条引用都至少有 1 个独立来源交叉印证吗？
2. 「不可用」评级有没有被引用？（一条都不能有）
3. 「低」评级的引用都在正文做了限定声明吗？
4. 图片 / 视频都做了反向搜了吗？

任一答否 → 修引用或删段落。

## 反例（什么时候不要用本 skill）

- 自己写主编观点段（不是引用，不需要核验，但**自己的观点必须基于已核验的事实**，写完观点段再回查事实段）
- 已经发布出去的稿件（事后发现错误用 `fact-checking` 的 Correction 分支）
